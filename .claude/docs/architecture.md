# Architecture

## Structure

```
/
├── scripts/              # CLI commands (flow-*.js) and hooks
│   ├── flow-*.js         # Core CLI modules (flow-start.js, flow-review.js, etc.)
│   ├── hooks/            # Hook system (three-layer: entry → core → adapter)
│   │   ├── entry/        # CLI-specific entry points (claude-code/)
│   │   ├── core/         # CLI-agnostic logic (task-gate.js, etc.)
│   │   └── adapters/     # CLI adapter layer
│   └── flow              # Main CLI entry point (shell script)
├── agents/               # Agent persona files (*.md) - 11 role definitions
├── lib/                  # NPM package modules (installer, bridge, etc.)
├── templates/            # NPM-distributed templates (shipped with package)
├── .workflow/            # Configuration and state
│   ├── config.json       # Main configuration (1600+ lines)
│   ├── state/            # Runtime state files (ready.json, request-log.md, etc.)
│   ├── agents/           # Review reference checklists (security.md, etc.)
│   ├── bridges/          # CLI bridge implementations (claude-bridge.js)
│   ├── models/           # Model registry (model-registry.json)
│   ├── templates/        # File and prompt templates (claude-md.hbs + partials)
│   ├── specs/            # Generated task specifications
│   ├── plans/            # Plan documents
│   ├── changes/          # Story/task change files
│   └── lib/              # Shared libraries
├── .claude/              # Claude Code integration
│   ├── commands/         # Slash commands (wogi-*.md → /wogi-* in CLI)
│   ├── skills/           # Installed skills (figma-analyzer)
│   ├── docs/             # Knowledge base documents
│   ├── rules/            # Coding rules (code-style, security, architecture, operations)
│   └── settings.local.json  # Hook registrations and permissions
├── CLAUDE.md             # Generated Claude Code instructions (from claude-md.hbs)
└── package.json          # NPM package definition
```

## Key Patterns

1. **CLI-Agnostic Core**: All logic in `.workflow/`, CLI-specific files generated by bridges
2. **State Files as Memory**: JSON/Markdown files in `.workflow/state/` persist context across sessions
3. **Skills System**: Modular skill packages with patterns, anti-patterns, learnings
4. **Hash-Based Task IDs**: `wf-XXXXXXXX` format for unique task identification
5. **Three-Layer Hook Architecture**: Entry (CLI-specific) → Core (logic) → Adapter (transform)
6. **Template-Driven Generation**: CLAUDE.md generated from `.workflow/templates/claude-md.hbs` + Handlebars partials

## Two Agent Directories

| Directory | Purpose | Files |
|-----------|---------|-------|
| `agents/` | 11 agent persona files (health checks, story writing, etc.) | orchestrator.md, developer.md, security.md, etc. |
| `.workflow/agents/` | Review reference checklists for wogi-review | security.md (OWASP checklist) |

## Two Model Systems

| System | File | Returns | Purpose |
|--------|------|---------|---------|
| Model Adapter | `flow-model-adapter.js` | String (model name) | Per-model prompt adjustments, learning |
| Model Registry | `flow-models.js` | Object `{name, info, source}` | Model listing, routing, cost tracking |

See `.claude/rules/architecture/model-management.md` for design rationale.

## Data Flow

```
User Command → /wogi-start (triage) → flow CLI / flow-*.js script → state files → response
                    ↓
            Request Triage → Route to appropriate command
                    ↓
            Explore Phase (3 parallel research agents)
                    ↓
            Spec Generation → Approval Gate (L1/L0)
                    ↓
            Implementation Loop (per acceptance criterion)
                    ↓
            Criteria Check → Wiring Check → Standards Check
                    ↓
            Quality Gates → Commit → Task Complete
```

## Config Structure

The main config (`.workflow/config.json`) is organized into these major sections:

| Section | Purpose |
|---------|---------|
| `enforcement` | Task gating, strict mode |
| `tasks` / `loops` | Execution loop behavior |
| `specificationMode` | Spec generation rules |
| `qualityGates` | Per-task-type quality requirements |
| `standardsCompliance` | Standards enforcement |
| `skills` / `skillLearning` | Skill system configuration |
| `hooks` | Hook system rules and targets |
| `planMode` | Explore phase and research agents |
| `research` | Zero-trust research protocol |
| `review` | Code review agents and multi-pass |
