# Prisma ORM Skill

## Overview
Prisma is a type-safe ORM for Node.js/TypeScript with auto-generated client from schema.

## Key Patterns (from reference project)
- Split schema: base.prisma + models/*.prisma merged via merge.ts script
- UUID primary keys with @default(uuid())
- Timestamps: createdAt @default(now()), updatedAt @updatedAt
- Soft delete via deletedAt DateTime?
- Table mapping with @@map("lowercase")
- PrismaService extends PrismaClient, implements OnModuleInit/OnModuleDestroy
- JSON fields for aggregated data (reactionsCount)

## Anti-Patterns
- Don't edit schema.prisma directly (it's generated by merge script)
- Don't forget to run prisma:merge before prisma:generate
- Don't use raw SQL when Prisma client methods suffice

## Conventions
- npm run prisma:merge -> prisma:generate -> prisma:migrate:new
- One model per .prisma file in prisma/models/
- Composite unique constraints with @@unique
- Cascade deletes on child relations
