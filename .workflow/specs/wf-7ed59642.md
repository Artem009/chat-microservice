# Spec: wf-7ed59642 — Add @mentions in Messages

## User Story
**As a** developer
**I want** @userId mention parsing in message content
**So that** mentioned users are notified via WebSocket events and mentions are persisted for querying

## Design Decision
Separate `Mention` Prisma model linking messageId + mentionedUserId. Mentions are extracted from message content at creation time by parsing `@<uuid>` patterns. A dedicated MentionModule handles CRUD. The CreateMessageController is extended to parse and create mentions as a side effect of message creation.

## Acceptance Criteria

### Scenario 1: Parse and create mentions on message creation
**Given** POST /api/message with content containing `@<userId>` patterns
**When** the message is created
**Then** Mention records are created for each unique @userId found in content
**And** only userIds that are active participants in the conversation are stored as mentions
**And** the sender is excluded from mentions (no self-mention)
**And** returns `{ data: message, mentions: mention[] }`

### Scenario 2: Notify mentioned users via WebSocket
**Given** a message with mentions is created
**When** mention records are stored
**Then** a `userMentioned` event is broadcast to the conversation room
**And** event payload: `{ messageId, conversationId, mentionedUserIds: string[] }`

### Scenario 3: List mentions for a message
**Given** GET /api/mention?messageId=<uuid>
**When** a valid messageId query param is provided
**Then** returns all mentions for that message
**And** returns `{ data: mentions[] }`

### Scenario 4: List mentions for a user
**Given** GET /api/mention?userId=<uuid>
**When** a valid userId query param is provided
**Then** returns all mentions where the user was mentioned
**And** returns `{ data: mentions[] }`

### Scenario 5: Unit tests
**Given** the mention module is created
**When** tests are run
**Then** mention parsing logic is tested (regex extraction, dedup, participant filtering, self-exclusion)
**And** controller + service methods are covered

## Implementation Steps

### Step 1: Prisma schema
- Create `prisma/models/mention.prisma` with: id, messageId, mentionedUserId, createdAt
- Add `mentions Mention[]` relation on Message model
- Run prisma:merge, prisma:generate, create migration

### Step 2: Mention module scaffold
- Create `src/mention/mention.module.ts`
- Create `src/mention/mention.service.ts` — thin Prisma wrapper with createMany(), findByMessage(), findByUser()
- Create `src/mention/mention-parser.ts` — pure function `parseMentions(content: string): string[]` that extracts unique @<uuid> patterns
- Register in AppModule

### Step 3: Controllers
- Create `src/mention/controllers/base.controller.ts` — injects MentionService
- Create `src/mention/controllers/list-mention.controller.ts` — GET /api/mention?messageId= or ?userId=
- Swagger decorators + @ApiQuery for both params

### Step 4: Integrate with message creation
- Modify `CreateMessageController` to:
  1. After creating message, call `parseMentions(dto.content)`
  2. Filter parsed userIds: only active participants in conversation (excluding sender)
  3. Create Mention records via MentionService.createMany()
  4. Broadcast `userMentioned` event if mentions exist
  5. Return `{ data: message, mentions }` (mentions array may be empty)
- Import MentionModule in MessageModule

### Step 5: Tests
- Create `src/mention/mention.spec.ts` — unit tests for parser + list controller
- Add mention tests to `src/message/message.spec.ts` for the create-message integration
