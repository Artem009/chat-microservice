# [wf-607119a1] Set up Prisma with PostgreSQL and split schema pattern

## Product Context
<!-- PIN: product-context -->
**Product**: chat-microservice-2
**Type**: Backend API microservice

---

## User Story
**As a** developer
**I want** Prisma ORM configured with the split schema pattern
**So that** database models are organized per-file and the schema is auto-merged

## Description
Install prisma and @prisma/client. Create the split schema structure: prisma/base.prisma (generator + datasource), prisma/models/ directory, prisma/merge.ts script. Create src/prisma/prisma.service.ts (extends PrismaClient, implements OnModuleInit/OnModuleDestroy) and src/prisma/prisma.module.ts. Add NPM scripts for merge, generate, migrate. Reference: comment-microservice/prisma/

## Acceptance Criteria

### Scenario 1: Schema merge works
**Given** base.prisma and models/*.prisma files exist
**When** `npm run prisma:merge` is executed
**Then** prisma/schema.prisma is generated combining all files

### Scenario 2: Prisma client generates
**Given** schema.prisma exists with valid models
**When** `npm run prisma:generate` is executed
**Then** @prisma/client is generated with typed models

### Scenario 3: PrismaService connects
**Given** DATABASE_URL is configured
**When** the app starts
**Then** PrismaService connects to PostgreSQL
**And** PrismaService disconnects on module destroy

### Scenario 4: PrismaModule exports service
**Given** PrismaModule is imported by a feature module
**When** a service injects PrismaService
**Then** the injection succeeds

## Technical Notes
- Install: `prisma` (dev), `@prisma/client`
- Create: `prisma/base.prisma`, `prisma/models/` (empty dir), `prisma/merge.ts`
- Create: `src/prisma/prisma.service.ts`, `src/prisma/prisma.module.ts`
- Add NPM scripts: `prisma:merge`, `prisma:generate`, `prisma:migrate:new`, `prisma:migrate:apply`, `prisma:migrate:status`, `prisma:short-cut:create`
- DATABASE_URL via env: `postgresql://user:pass@localhost:5432/dbname`
