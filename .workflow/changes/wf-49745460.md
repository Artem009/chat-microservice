# [wf-49745460] Align tsconfig module system with reference

## Product Context
<!-- PIN: product-context -->
**Product**: chat-microservice-2
**Type**: Backend API microservice

---

## User Story
**As a** developer
**I want** the tsconfig aligned with the reference project conventions
**So that** module resolution is consistent across microservices

## Description
Current tsconfig uses "nodenext" module/moduleResolution. Reference uses "commonjs". Evaluate whether to switch to commonjs for cross-project consistency or keep nodenext. Also add "strict": true to match reference.

## Acceptance Criteria

### Scenario 1: Module system decided
**Given** the developer reviews the trade-offs
**When** a decision is made
**Then** tsconfig.json is updated accordingly
**And** the project builds and tests pass

## Technical Notes
- Current: `"module": "nodenext"`, `"moduleResolution": "nodenext"`
- Reference: `"module": "commonjs"` (no explicit moduleResolution)
- Reference also has `"strict": true`
- Decision needed: nodenext is newer/ESM-compatible, commonjs is reference standard

## Dependencies
- None

## Complexity
Low - Config change with testing verification
