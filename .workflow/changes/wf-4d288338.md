# [wf-4d288338] Add global ValidationPipe with class-validator

## Product Context
<!-- PIN: product-context -->
**Product**: chat-microservice-2
**Type**: Backend API microservice

---

## User Story
**As a** developer
**I want** automatic DTO validation on all incoming requests
**So that** invalid payloads are rejected before reaching controllers

## Description
Install class-validator and class-transformer. Configure global ValidationPipe in main.ts with whitelist (strip unknown fields), forbidNonWhitelisted (reject unknown fields), and transform (auto-convert types). Reference: comment-microservice/src/main.ts

## Acceptance Criteria

### Scenario 1: Valid payload accepted
**Given** a DTO with @IsString() @IsNotEmpty() decorators
**When** a valid payload is sent
**Then** the request proceeds to the controller

### Scenario 2: Invalid payload rejected
**Given** a DTO with validation decorators
**When** an invalid or empty payload is sent
**Then** a 400 Bad Request is returned with validation error details

### Scenario 3: Unknown fields stripped
**Given** whitelist is enabled
**When** a payload contains unexpected fields
**Then** the request is rejected (forbidNonWhitelisted)

## Technical Notes
- Install: `class-validator`, `class-transformer`
- Add `app.useGlobalPipes(new ValidationPipe({ whitelist: true, forbidNonWhitelisted: true, transform: true }))` in main.ts
